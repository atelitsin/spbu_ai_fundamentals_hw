## Прогноз справедливой цены и поиск выгодных предложений на рынке смартфонов по данным Авито

### Abstract

В проекте рассматривается задача оценки рыночной стоимости б/у смартфонов по объявлениям Авито для города Санкт-Петербург. Цель проекта состоит в построении модели, которая по данным объявления предсказывает справедливую цену смартфона и позволяет выявлять выгодные предложения, где запрашиваемая цена заметно ниже предсказанной моделью. В качестве основного подхода будут использоваться классические методы машинного обучения, а качество оцениваться по MAPE.

### Goal and Metrics

* Цель - предсказание цены смартфона и поиск выгодных предложений на рынке б/у смартфонов.
* Дополнительная прикладная цель - ранжирование объявлений по степени выгодности покупки.
* Основная метрика оценки - MAPE.
* Дополнительная метрика для интерпретации ошибок - MAE в рублях.

### Data

Данные получены путем парсинга объявлений о продаже смартфонов по городу Санкт-Петербург на Авито.

Основные признаки:

1. Производитель смартфона (brand)
2. Модель смартфона (model)
3. Объем встроенной памяти
4. Объем оперативной памяти
5. Состояние устройства
6. Цвет
7. Наличие комплекта, коробки или чека, если это указано в объявлении
8. Текст заголовка и описания
9. Цена объявления
10. Дата публикации объявления
11. Ближайшая станция метро
12. Количество фотографий
13. Данные о продавце.

Перед обучением данные будут очищены от дублей, пустых и некорректных объявлений.

### Validation Draft

Проверка предсказаний модели будет осуществляться на отложенной выборке. Чтобы снизить риск утечки, разбиение планируется делать по времени публикации: более ранние объявления будут использоваться для обучения, а более поздние - для тестирования. Это позволит приблизить оценку качества к реальному сценарию применения модели на новых объявлениях. Основная метрика оценки - MAPE, которая измеряет среднюю абсолютную процентную ошибку между предсказанными и фактическими ценами смартфонов.

### Approach

В качестве baseline-модели будет использоваться линейная регрессия.

Полный список признаков для модели:

1. brand
2. model
3. memory
4. ram
5. condition
6. color
7. has_box
8. has_charger
9. has_receipt
10. district
11. metro
12. photos_count
13. description_len
14. description_feature_1

Для табличных признаков будут использоваться кодирование категориальных переменных, обработка пропусков и отбор выбросов. Для текста планируется сравнить простые признаки на основе TF-IDF и модели без текстовых признаков, чтобы понять вклад описания и заголовка в качество прогноза.

После baseline будут протестированы более сильные классические модели, такие как *TBD*, если они покажут улучшение качества.

### Training

Помимо baseline-модели, планируется обучить несколько альтернативных моделей с разными наборами признаков и гиперпараметров. Сравнение будет включать:

1. Linear Regression / Ridge
2. CatBoostRegressor
3. LightGBMRegressor или аналогичную градиентную модель

Для каждой модели будет выполнен подбор гиперпараметров на валидационной выборке. Отдельно будет проведен анализ ошибок на дорогих и дешевых смартфонах, а также по брендам и состоянию устройства.

### Validation

После обучения модели будут сравниваться по MAPE и MAE на тестовой выборке. Дополнительно будет выполнен residual analysis: проверка распределения ошибок, сравнение предсказаний и фактических цен, анализ случаев сильного недооценивания и переоценивания. Это поможет понять, в каких сегментах рынка модель работает лучше или хуже.

### Model Selection

Финальная модель будет выбрана по сочетанию качества на тестовой выборке, устойчивости на разных сегментах данных и удобства интерпретации. Основные риски подхода:

1. Неполные или шумные данные из объявлений.
2. Сильная зависимость цены от редких характеристик, которые могут быть плохо представлены в данных.
3. Изменение рынка со временем, из-за которого модель может устаревать и требовать регулярного переобучения.

### Deploy

Используемая в проекте модель может быть использована для поиска выгодных предложений в реальном времени. Точность предсказания зависит не только от качества модели, но и от актуальности данных, на которых производилось обучение. Для более точных предсказаний рекомендуется использовать данные не более чем недельной давности.
